# What documents to maintain?

- For every module the `docs` folder must contain a `Requirements.md` file.
- The `Requirements.md` file must document the high-level functional features provided by that module.
- Keep requirements high level and focused on what the module does, not how it is implemented.
- Whenever a new feature is added to a module, update that module's `Requirements.md` file in the same change.
